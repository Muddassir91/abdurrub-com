# 3D Rose — STL + CNC package

A parametric 3D rose head, designed so a CNC router can carve it from a single
wooden blank in two passes (rough then finish). Ship-ready files for both
*looking at* the model and *machining* it.

## Contents

```
cnc-rose/
├── rose.stl              <- the 3D model (binary STL, 35,216 triangles)
├── preview.html          <- browser preview (three.js)
├── rose_roughing.nc      <- G-code: 6 mm flat end mill, Z-stepdown raster
├── rose_finishing.nc     <- G-code: 3 mm ball nose, surface-following raster
├── setup_sheet.md        <- stock, tools, zeros, feeds, run order, safety
├── generate_rose.py      <- source: parametric rose -> STL
├── generate_gcode.py     <- source: STL -> G-code
└── README.md             <- this file
```

## Preview the model

The STL viewer needs to fetch `rose.stl` over HTTP, so don't open the file
directly with `file://`. From this folder:

```bash
python3 -m http.server 8000
```

Then visit <http://localhost:8000/preview.html>. Drag to orbit, scroll to zoom,
right-drag to pan. The dashed teal box shows the 100 × 100 × 40 mm stock blank
the CNC will cut from; the X/Y/Z axes mark the work-coordinate origin.

You can also drag `rose.stl` straight into any STL viewer (Microsoft 3D Viewer,
Windows File Explorer thumbnail, MeshLab, PrusaSlicer, Bambu Studio, Fusion 360,
<https://www.viewstl.com/>, etc.).

## Cut it on a CNC

Read **`setup_sheet.md`** before doing anything. Short version:

1. Square a **100 × 100 × 40 mm hardwood blank**, clamp it on its outer 12 mm rim.
2. Set **G54** to the **top-centre** of the blank, with **Z0 on the top face**.
3. Install **T1 — 6 mm flat end mill**. Run `rose_roughing.nc`. (~10 min)
4. Tool change to **T2 — 3 mm ball nose**. Re-touch Z0 on the stock top.
5. Run `rose_finishing.nc`. (~40 min)
6. Sand the petal edges and pop the part off the spoilboard.

Always **air-cut once** with a +30 mm Z offset to confirm the toolpath stays
inside its bounds.

## Regenerate everything

```bash
python3 generate_rose.py     # writes rose.stl
python3 generate_gcode.py    # writes rose_roughing.nc and rose_finishing.nc
```

Both scripts are pure Python 3 (no dependencies). Edit them to change petal
geometry, tooling, feeds, or stock dimensions.

## Honest caveats

- The G-code is generated by a custom Python height-map → raster pipeline, not
  by an industrial CAM package. It's correct in form (G21/G90/G54, no
  extra-deep moves, retracts between passes) but **always verify it in your
  sender's preview and air-cut first** before committing wood to it.
- The model is a sculpture, not a botanical-accurate rose. Tune the `layers`
  recipe in `generate_rose.py` to taste.
- Petal tips are 0.6 mm thick. Use **hardwood** — soft pine will splinter.
